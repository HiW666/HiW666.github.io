本教程使用的单片机为stm32f103c8t6

简单介绍

stm32系列单片机
有68个可屏蔽中断通道，包含EXTI、TIM、ADC、USART、SPI、I2C、RTC等多个外设
这种中断源统一由NVIC统一来进行管理。
为什么要NVIC进行管理，因为NVIC可以有中断优先级。

NVIC介绍

NVIC里面分为抢占优先级和响应优先级

分组方式 抢占优先级 响应优先级
分组0 0位，取值为0 4位，取值为0~15
分组1 1位，取值为0~1 3位，取值为0~7
分组2 2位，取值为0~3 2位，取值为0~3
分组3 3位，取值为0~7 1位，取值为0~1
分组4 4位，取值为0~15 0位，取值为0

抢占优先级相当是十位数
响应优先级相当是个位数

比如
中断 A：抢占优先级 0，响应优先级 0；
中断 B：抢占优先级 1，响应优先级 0
中断 C：抢占优先级 1，响应优先级 1

A和B触发，那么A会停止，等B完成了A才会继续触发
当然A和C也是同理
但是B和C同时触发
C会运行，但是B不会运行。
仅抢占优先级不同时，才会发生中断嵌套
响应优先级仅影响同抢占优先级中断的响应顺序，无嵌套关系。

EXTI介绍

EXTI（Extern Interrupt）外部中断

EXTI可以监测指定GPIO口的电平信号，当其指定的GPIO口产生电平变化时，EXTI将立即向NVIC发出中断申请，经过NVIC裁决后即可中断CPU主程序，使CPU执行EXTI对应的中断程序

支持的触发方式：上升沿/下降沿/双边沿/软件触发

支持的GPIO口：所有GPIO口，但相同的Pin不能同时触发中断

通道数：16个GPIO_Pin，外加PVD输出、RTC闹钟、USB唤醒、以太网唤醒

触发响应方式：中断响应/事件响应

因为我们希望的是使用GPIO来触发所以要用EXTI

然后关于EXTI的设计
我们都知道GPIO的设计是分成A组，B组，C组。然后再划分引脚号
但是EXTI反人类的是，他不是安装A组，B组设计的。

我们可以看到图片，他是引脚号为组，里面再划分A，B，C，D等
这很重要，下面要考的。

初始化EXTI

参数 数据类型 作用 选项
EXTI_InitTypeDef* EXTI_InitStruct 结构体 初始化EXTI —
返回值 无 无 无

结构体内部参数：

EXTI_InitTypeDef EXTI_InitStructure; 定义结构体

名称 参数 作用
EXTI_InitStructure.EXTI_Line EXTI_Line0 //对应所有GPIO的Pin0
EXTI_Line1 //对应所有GPIO的Pin1
…
EXTI_Line14 //对应所有GPIO的Pin14
EXTI_Line15 //对应所有GPIO的Pin15		 选择外部中断线
EXTI_InitStructure.EXTI_LineCmd ENABLE/DISABLE 使能/关闭中断线
EXTI_InitStructure.EXTI_Mode EXTI_Mode_Interrupt 中断模式
EXTI_Mode_Event 事件模式		 选择中断模式
EXTI_InitStructure.EXTI_Trigger EXTI_Trigger_Rising 上升沿触发
EXTI_Trigger_Falling 下降沿触发
EXTI_Trigger_Rising_Falling 双边沿触发		 触发模式

关于事件模式：
中断模式：引脚触发 → 产生中断请求 → CPU 暂停主程序 → 执行中断服务函数
按键触发控制
传感器电平变化
所有需要执行代码逻辑的场景
事件模式：引脚触发 → 产生硬件事件信号 → 不进入中断函数
低功耗休眠唤醒 仅用于硬件触发（如唤醒 CPU、触发 DMA）
硬件事件联动（不常用）不占用CPU，不执行代码。

配置NVIC

NVIC的组设置

参数 数据类型 作用 选项
uint32_t NVIC_PriorityGroup uint32_t 中断优先级分组（看下方） NVIC_PriorityGroup_0
NVIC_PriorityGroup_1
NVIC_PriorityGroup_2
NVIC_PriorityGroup_3
NVIC_PriorityGroup_4
返回值 无 无 无

中断优先级分组的作用就是决定抢占优先级和响应优先级的数量

优先级分组 抢占优先级位数 响应优先级位数 取值范围
Group_0 0 位 4 位 抢占：0
响：0-15
Group_1 1 位 3 位 抢占：0-1
响：0-7
Group_2 2 位 2 位 抢占：0-3
响：0-3
Group_3 3 位 1 位 抢占：0-7
响：0-1
Group_4 4 位 0 位 抢占：0-15
响：0
